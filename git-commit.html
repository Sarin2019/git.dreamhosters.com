The git commit command

The git commit command commits files in the staging area to the project history. Git will never change a commit under you specifically tell it to do so.

Running the command

Run the following command to commit your staged files while at the same time adding a small message.

[server]$ git commit -am "message here"

The -a flag says to add all changes in the working directory. These changes only include modifications to tracked files.

The -m flag is so you can add a message.

View files in the commit

To view what is currently in the commit, do the following:

  1. After you commit you'll see an ID#. For example: 
    [server]$ git commit -m "2nd upstream commit"
[master 6c0ae3a] 2nd upstream commit
 1 file changed, 1 insertion(+)
 create mode 100644 upstreamfile2.txt
  2. Copy the ID#. In this example it is 6c0ae3a.
  3. Run the following command with that ID to view what is in the commit. 
    [server]$ git diff-tree --no-commit-id --name-only -r 6c0ae3a
upstreamfile2.txt
