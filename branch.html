Overview

A branch is an alternate version of the main branch of a project (usually the master branch).

When you create a branch, you create a copy of the master branch. You can then edit this as you like without affecting the master branch.

This page explains how to create a branch and push it to a repository on github.com.

View all branches

You can view all branches in a repo by running the git branch command:

[server]$ git branch
* master

Since only the 'master' branch shows, this informs you that there are no other branches in this repository.

Create a branch

Run the following to create a branch. You can name it anything you like.

[server]$ git branch testbranch

Run git branch to confirm it's been added.

[server]$ git branch
* master
  testbranch

You can see the 'master' branch is still selected. You must now switch to the new branch.

Switch to the new branch

Run the git checkout branchname command to switch to the new repo.

[server]$ git checkout testbranch

[server]$ git checkout testbranch
Switched to branch 'testbranch'

Then run git branch to confirm you're now on the new branch.

[server]$ git branch
  master
* testbranch

Push your branch to the remote repo on github.com

  1. In your new branch, edit any files you wish to push to the remote repository.
  2. Add and commit the files with: 
    [server]$ git commit -am "message here"
  3. View the files in your commit before pushing to confirm everything looks good. To do this run: 
    [server]$ git diff-tree --no-commit-id --name-only -r commitID#
  4. Push to the remote repository with: 
    [server]$ git push -u origin branchname
    'origin' is where you want it to go. Run git remote -v to confirm where you're pushing to.
  5. Visit your forked repository at github.com.
  6. From the top left BRANCH dropdown, choose your new branch you just committed.
  7. Create a pull request from that branch. Below the BRANCH dropdown (which should have your new branch selected already), click the 'New Pull Request' button.
  8. The pull request now shows on the primary (master) repository.

Make an addition to a pull request from a specific branch

The following shows how to update your local repository from a branch someone else pushed to the main remote repository.

Once updated you can edit and push your changes back to the remote repository to show as an addition to the pull request.

  1. Run the following to update your local repo: 
    [server]$ git remote update && git checkout master && git pull --ff-only origin
    This syncs your local repo with the upstream repo. This includes pulling in the branch where the pull request is coming from.
  2. Checkout the branch: 
    [server]$ git checkout branchname
    You can find 'branchname' on the top right of the pull request in github.com.
  3. You should now see the changes in your local files since you're on this new branch.
  4. Edit any files you like.
  5. Commit the file: 
    [server]$ git commit -am "commit message"
  6. Push your changes: 
    [server]$ git push
  7. The changes should now show as an addition to the pull request.
  8. The admin of the github.com repo must now review the pull request and merge any changes.
