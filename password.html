Overview

It can get annoying to enter your user:pass every time you push to github.com.. There are two options to save your password

Insecure permanent method

This method should only be used if you are the only one who has access to your computer and repository. This is because the password is stored in plain text.

In your repository navigate to the /.git/config file.

Open this file and edit the [remote "origin"] url line:

[remote "origin"]
url = https://username:password@github.com/username/repositoryname.git

So the only part you add is the username:password before the @ sign.

If you run git remote -v you can see your user:password. So this is not a secure method to use.

Secure temporary method

Run the following in your repository directory:

[server]$ git config --global credential.helper 'cache --timeout=3600'

Change the timeout= to any amount of seconds you like.

This caches your password after you enter it the first time for the number of seconds you set.

It also creates a file named .gitconfig in your shell user's directory with the following content:

[user]
        name = John Smith
        email = john.smith@example.com
[credential]
        helper = cache --timeout=3600
