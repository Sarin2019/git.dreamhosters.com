Overview

This page details a very basic git workflow. This is just a general idea of how you may interact with most projects.

Workflow

  1. Create or clone a repository to your local computer.
    • If you're opening a previously saved project (on your local computer) from a remote shared repo, make sure to download any updates from the remote repo. You can do this using git pull, or git fetch followed by git merge.
  2. Edit files in that repository.
  3. Add them to your staging area using git add.
  4. Commit the files using git commit.
  5. If you're using an external site such as github.com, push those changes to the repository using git push.
