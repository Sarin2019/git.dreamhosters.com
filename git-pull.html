Overview

The git pull command is a quick way to run git fetch followed by git merge.

Usage

You can pull in remote changes in two ways:

If the repository is on github.com

[server]$ git pull https://github.com/repositoryname.git

If you've added a remote repository using git remote

[server]$ git pull remotename branchname

For example:

[server]$ git pull upstream master
From https://github.com/upstreamrepository
 * branch            master     -> FETCH_HEAD
Processing '/etc/joe/editorrc'...Processing '/etc/joe/ftyperc'...done
done

File /home/mygituser/gitrepos/gittestrepo/.git/MERGE_MSG saved
Merge made by the 'recursive' strategy.
 upstreamfile3.txt | 0
 upstreamfile4.txt | 0
 2 files changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 upstreamfile3.txt
 create mode 100644 upstreamfile4.txt

This will open a commit message prompt. Enter a simple description of what you're committing. Then click CTRL + K + X to save and exit.

Errors

If your local repository has information that is not on the rmeote repository, you'll see a merge conflict. Take the following steps to resolve.

  1. Run git status to view which files have issues.
  2. Open each of those files in an editor.
  3. You'll see two versions of the section with an issue. Choose which version you want to keep. Delete the code from the other version including any formatting tags like ==== or >>> or <<<.
  4. Add your changes using git add.
  5. Commit your changes.
  6. Push changes to your origin master branch.
  7. Try running the git pull again and it should work.
