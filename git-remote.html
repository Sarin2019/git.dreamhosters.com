Overview of the git remote command

The git remote command allows you to manage tracked repositories. For example, you can add an upstream repo to pull data from.

View tracked repos

Run git remote -v to view tracked repositories: 

[server]$ git remote -v
origin  https://github.com/gittestrepo.git (fetch)
origin  https://github.com/gittestrepo.git (push)

The -v (verbose) flag displays all tracked repositories.

Add an upstream repo

You can also add an upstream repository to pull data from. First, find the URL of the repository on github.com or from the owner. Then runn the following to add it:

[server]$ git remote add upstream https://github.com/upstreamrepository.git

If you run git remote -v now, you'll see the newly tracked repo: 

[server]$ git remote -v
origin  https://github.com/gittestrepo.git (fetch)
origin  https://github.com/gittestrepo.git (push)
upstream        https://github.com/upstreamrepository (fetch)
upstream        https://github.com/upstreamrepository (push)

You can now run git fetch upstream master to update your remote repository from the upstream repository. 'master' must be the branch you wish to fetch. This is often master.

The word 'upstream' could be anything. This is just a common titled used.

Delete remote branch

To delete a branch run the following:

[server]$ git remote rm upstream

Make sure 'upstream' is whatever remote repo you wish to remove.
