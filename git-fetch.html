The git fetch command

[server]$ git fetch upstream master
remote: Counting objects: 3, done.
remote: Compressing objects: 100% (2/2), done.
remote: Total 3 (delta 1), reused 0 (delta 0), pack-reused 0
Unpacking objects: 100% (3/3), done.
From https://github.com/upstreamrepository
 * branch            master     -> FETCH_HEAD
   1aa3afb..e3cd350  master     -> upstream/master

The git fetch command says to go back to wherever you cloned from (or have otherwise set up as a place to fetch from) and find "new stuff someone else added or changed or removed".

Those changes are copied into your history only. They are not applied to your local repository yet.

Once you run git fetch, check your log history to view what exactly was 'fetched'.

[server]$ git log --all
commit e3cd350ba5c225bb63ed075ccd9de765bb8c8c3a
Author: username 
Date:   Mon Oct 10 18:42:48 2016 -0700

    Create upstreamfile.txt

In this case, only a single file is new in the upstream repo.

You now must 'merge' these changes into your local repository.
